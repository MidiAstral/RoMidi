How it works

4 steps to play your own songs

RoMidi scans your executor workspace automatically every time you press RELOAD. No config files to edit, no code to write.

01
Find your executor workspace
Every executor stores a "workspace" folder on your device. The location is different for each executor. Delta, ArceusX, Fluxus and others each have their own path. See the full list in the section below.
02
Create a top-level folder
Inside the workspace, create a folder with any name you like - "MySongs", "RoMidi", "Piano", literally anything. This folder is just a container. RoMidi scans every folder in the workspace root automatically.
03
Add subfolders as categories
Inside your top-level folder, create subfolders. Each subfolder name becomes a category in the RoMidi library. A subfolder named "Anime" appears as the "Anime" category. You can create as many categories as you want.
04
Put .mid files inside
Drop your .mid or .smf files into the category subfolders. The file name without the extension becomes the song name in the library. Press RELOAD in RoMidi and all your songs appear instantly.

File Rules

What RoMidi accepts and rejects

A few strict rules that determine whether your songs show up or not.

✅
Accepted file formats
RoMidi only reads files with the extension .mid or .smf. Both are standard MIDI formats. Any other file type - .mp3, .wav, .txt - is completely ignored.
song.midsong.smf
❌
Files in the wrong level
MIDI files placed directly in the workspace root or directly inside the top-level folder are ignored. They must be inside a category subfolder. The required depth is: workspace / folder / category / song.mid
workspace/song.midMySongs/song.mid
📁
Multiple top-level folders work
You can have more than one top-level folder in your workspace. RoMidi scans all of them and merges categories with the same name. Songs from "Pack1/Anime/" and "Pack2/Anime/" both appear together under the "Anime" category.
Multiple root folders supported
📝
Song name = file name
The display name in the library is taken directly from the file name, with the extension removed. "Moonlight Sonata.mid" becomes "Moonlight Sonata". Rename the file to change how it appears in RoMidi.
Rename file to rename song
⚠️
Duplicate file names are deduplicated
If two files have the same name within the same category, only the first one found is kept. RoMidi deduplicates by file name per category to prevent confusion. Give your songs unique names to avoid this.
Two "Sparkle.mid" in same folder
🔄
Add songs without rejoining
You do not need to leave or rejoin the Roblox game at all. Add or move files in the workspace folder on your device, go back to RoMidi inside the game, press RELOAD, and your new songs appear in the library immediately.
Press RELOAD to refresh anytime
FAQ

Common questions

The most frequent problems and answers for local MIDI setup.

My songs are not showing up after pressing RELOAD. What is wrong?
⌄
Check the folder depth. MIDI files must be exactly three levels deep: workspace root, then a named folder, then a category subfolder, then the .mid file. If a file is at any other level it will not be detected. Also confirm the file ends in .mid or .smf - other extensions are rejected. After fixing the structure press RELOAD in RoMidi.
Where exactly do I find the workspace folder on my phone?
⌄
On Android, open your device file manager app and browse to your internal storage root. You should see a folder with the executor name such as "Delta", "Arceus", "Fluxus". Open that folder and you will find the "workspace" folder inside. On iOS with Delta, use the Delta app's built-in file browser or access it through the iOS Files app under the Delta section.
Can I have more than one category subfolder?
⌄
Yes, you can create as many category subfolders as you want. Each one becomes its own labeled category in the Local tab of the RoMidi library. Common examples are Anime, Classical, Pop, Game OST, Sad, and Vibes. There is no limit on how many categories you can have.
Does the name of the top-level folder matter?
⌄
No. The top-level folder can be named anything at all. "RoMidi", "MySongs", "Piano", "XYZABC" - it does not matter. RoMidi scans every single folder found in the workspace root without filtering by name. Only the subfolder names inside it matter because they become your category labels in the library.
Does Local MIDI work differently on mobile vs PC?
⌄
The only difference is where the workspace folder lives on your device. The folder structure inside is completely identical on both platforms. Once your files are in the correct subfolders, pressing RELOAD in RoMidi loads them exactly the same way on PC and mobile - no extra steps needed.
Where can I download MIDI files for my favorite songs?
⌄
Search for "[song name] MIDI download" on sites like MuseScore (musescore.com), Bitmidi (bitmidi.com), FreeMidi (freemidi.org), or Musescore's free sheet music section. Download the .mid file and place it inside the correct category subfolder in your executor workspace. Most piano sheet music sites also offer MIDI exports alongside the sheet music.
Can I favorite my local songs the same way as preset songs?
⌄
Yes. Once a local song is loaded in the library list, tap the heart icon on its row to add it to Favorites. Local favorites are saved alongside preset favorites and appear at the top of the library panel with a pink border so you can access them instantly without having to navigate back to the Local tab.
Tips

Get the most out of local songs

Small habits that make managing your collection much smoother.

TIP 01
Use clean file names
The file name is exactly what appears in the library. Avoid symbols like slashes, colons, asterisks or question marks. Use spaces or hyphens instead - for example "River Flows in You.mid" works perfectly.
TIP 02
Organize by genre
Create separate subfolders for Anime, Classical, Pop, Game OST, etc. RoMidi displays each subfolder as its own dropdown category so you can find songs quickly instead of scrolling through one long list.
TIP 03
Combine with Favorites
Tap the heart on a local song row after loading it to favorite it. Local favorites are pinned at the top of the library together with preset favorites, so your go-to songs are always one tap away regardless of which tab you are on.
TIP 04
Turn Auto BPM on
Most MIDI files have tempo change markers built in. Enable Auto BPM before playing a local file and RoMidi follows those markers automatically, so the song plays at the correct original speed without any manual input from you.
TIP 05
Use multiple top folders
You can keep separate song packs in different top-level folders - "Pack1", "Pack2", "Favorites". RoMidi scans all of them and merges matching category names so everything appears in one unified library view.
TIP 06
RELOAD after every change
RoMidi caches the song list in memory after the first scan. Whenever you add, remove, or rename files in the workspace, press RELOAD in RoMidi to pick up all the changes without leaving Roblox.
