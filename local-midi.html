How Local MIDI Works

RoMidi can read MIDI files that you place inside your executor's workspace folder. When you run the script, it scans that folder for any subfolders and their contents, turning each subfolder into a browseable song category inside the in-game Song Browser.

You do not need any tools, converters, or special setup. Just copy your .mid files into the right folders and the script finds them automatically.

ⓘ
How the scan path works RoMidi looks inside your executor's workspace/ folder. Everything is organized by a top-level folder you name yourself, with song files living inside subfolders of that folder. The subfolder names become the category names you see in the dropdown inside RoMidi.
📁

Your top-level folder in workspace

You can name this anything. For example MyMidi or Piano Songs. This is the root folder RoMidi scans inside.

🗀

Subfolders inside it become categories

Each subfolder you create inside your top-level folder becomes one selectable category in the Local dropdown of the Song Browser. Name them anything — for example Anime, Pop, Classical.

🎵

MIDI files inside those subfolders are the songs

Any .mid or .smf file you put inside a subfolder shows up as a playable song in that category inside RoMidi.

2

Step 1 — Find Your Executor Folder

Every script executor has a folder on your computer where it stores files. This is where you put your MIDI files so the script can read them. The location depends on which executor you use.

  • 1
    Open your executor Launch your Roblox executor application on your PC.
  • 2
    Find the workspace folder Look for a folder called workspace inside your executor's installation directory. Most executors place it at a path like C:\Users\YourName\executor-name\workspace\. Some have a button in the UI labeled "Open Workspace" or a folder icon.
  • 3
    Open it in File Explorer Open that workspace folder in Windows File Explorer (or your file manager on other systems). This is where everything goes.
💡
Quick tip Many executors have a shortcut button to open the workspace folder directly. Check the executor's settings, file panel, or right-click menu for an "Open Folder" or "Workspace" option.

Step 2 — Understand the Folder Structure

The key rule is simple: songs must always live inside a subfolder, never directly inside the root folder. That subfolder name is what becomes the category name in RoMidi.

▶
The required structure workspace / YourFolderName / SubfolderName / song.mid
The top-level folder and all subfolder names can be anything you want. The only requirement is that your .mid files are inside a subfolder, not loose in the root.
structure workspace/ (your executor's workspace folder) MyMidi/ <-- your top-level folder, any name Anime/ <-- category: "Anime" unravel.mid silhouette.mid gurenge.mid Pop/ <-- category: "Pop" shape-of-you.mid blinding-lights.mid Classical/ <-- category: "Classical" moonlight-sonata.mid fur-elise.mid

In the example above, pressing the Local dropdown inside RoMidi would show three categories: Anime, Pop, and Classical. Selecting any of them shows the MIDI files inside.

⚠
Do not put songs directly in the root folder If you place a .mid file directly inside MyMidi/ without a subfolder, RoMidi will not see it. Every song needs to be inside at least one subfolder.

4

Step 3 — Add Your MIDI Files

Once you understand the structure, adding songs is as simple as copying files into the right subfolder. Here is the full process from scratch.

  • 1
    Create your top-level folder Inside your executor's workspace folder, create a new folder and name it anything you like. For example: MyMidi, Piano, or Songs.
  • 2
    Create one or more category subfolders Inside that top-level folder, create subfolders for each category of songs you want. For example Anime, Pop, OST, My Favorites. Each one becomes a separate category tab inside RoMidi.
  • 3
    Copy your MIDI files into the subfolders Drag and drop (or copy and paste) your .mid or .smf files into the correct subfolder. You can put as many files as you want in each category. File names become the song names shown in RoMidi.
  • 4
    Run RoMidi or press RELOAD If RoMidi is already running, press the RELOAD button in the utility row. If you are starting fresh, just run the script and it will pick up all your files automatically.
  • 5
    Open the Song Browser and switch to Local Click LIST in the RoMidi header, then click the Local dropdown. Your folder names appear as categories and your MIDI files appear as songs.

5

Naming Rules and Tips

You have a lot of freedom with names, but a few things are worth knowing to keep everything organized.

  • ✓ Folder names can be anything — spaces, numbers, special characters all work. Whatever you name the subfolder is exactly what shows up as the category label in RoMidi.
  • ✓ Song file names can be anything — the file name (without the .mid extension) is used as the song title inside the browser list.
  • ✓ You can create as many subfolders as you want — there is no hard limit. If you have 20 categories, all 20 show up in the Local dropdown.
  • ✓ You can add or remove songs at any time — just copy new files in or delete old ones, then press RELOAD inside RoMidi to refresh the list.
  • ⚠ Songs must be inside a subfolder — do not place .mid files directly in your top-level folder. RoMidi only reads files that are one level deeper inside a named subfolder.
  • ⚠ Only .mid and .smf files are supported — other formats like .mp3, .wav, or .musicxml will not load. Convert them to MIDI first using a tool like MuseScore or GarageBand.
💡
Keep song file names clean Since the file name becomes the title shown in RoMidi, try to use readable names like Moon River.mid instead of mr_v2_final_FINAL.mid. Short, clear names are easier to search and tap in-game.

Reloading the Library

If you add, remove, or rename MIDI files while RoMidi is already running, you do not need to re-execute the script. Just press the RELOAD button in the utility row at the bottom of the main panel.

  • 1
    Add or remove files in File Explorer Copy new .mid files into a subfolder, delete old ones, or create entirely new category subfolders.
  • 2
    Press RELOAD inside RoMidi The RELOAD button is in the bottom utility row of the main panel. Tapping it clears the current library cache and rescans your workspace folders from scratch.
  • 3
    A toast confirms the refresh A green "Library reloaded!" toast appears when the scan is complete. Your new songs and categories are immediately visible in the Local dropdown.

Supported File Types

RoMidi's built-in MIDI parser supports the following file formats.

accepted .mid Standard MIDI File — the most common format .smf Standard MIDI File — same format, different extension
⚠
Other formats will not work Files like .mp3, .wav, .flac, .musicxml, or .mxl are not MIDI files and cannot be read by RoMidi. You need to convert them to .mid format first. Free tools like MuseScore, GarageBand, or online converters can help.

The parser supports:

  • ✓Multi-track MIDI files (Format 0 and Format 1)
  • ✓Files with embedded tempo markers (used by Auto BPM)
  • ✓Running status byte compression
  • ✓Automatic cleanup of orphaned note-off events to prevent stuck notes
  • ✓Standard division / ticks-per-quarter-note timing
